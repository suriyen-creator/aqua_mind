## 1. เป้าหมายที่ต้องส่งให้สำเร็จ

ส่ง AquaMind ในสถานะ **Technical Prediction Prototype** ที่พิสูจน์ได้ว่า:

1. XGBoost รับ Feature และสร้าง Probability ได้จริง
2. SHAP อธิบายผลของ XGBoost ต่อคำขอได้จริง
3. Weather/Ocean forecast และ Sentinel-2 ของบางแสนเชื่อมต่อจริง
4. ระบบจัดการกรณีไม่มีภาพ/ข้อมูลไม่พอโดยไม่สร้างเปอร์เซ็นต์ Live ขึ้นเอง
5. ผู้ใช้เห็น Data age, แหล่งข้อมูล ข้อจำกัด และแนวทางตอบสนอง

คำว่า **Prediction** ในงานส่งวันนี้หมายถึงการสาธิต Model inference ด้วยข้อมูลสังเคราะห์ ไม่ใช่การรับรองความแม่นยำในพื้นที่จริง

## 2. สิ่งที่ห้ามทำเพราะจะทำให้งานไม่น่าเชื่อถือ

- ห้ามสร้าง Ground truth ปลอมแล้วเรียกว่าข้อมูลจริง
- ห้ามนำ Weather/Ocean API มาใช้เป็น Label
- ห้ามอ้าง Accuracy, Precision หรือ Recall จากข้อมูลสังเคราะห์ว่าเป็นผลภาคสนาม
- ห้ามแสดงเปอร์เซ็นต์ Live บางแสนเมื่อโมเดลยังไม่ผ่าน Validation
- ห้ามเรียก Sentinel-2 ว่าเป็นการยืนยันชนิดหรือความเป็นพิษของแพลงก์ตอน
- ห้ามเพิ่มโมเดลใหม่ที่ไม่ได้ทดสอบในช่วงก่อนส่ง

## 4. รูปแบบผลทำนายที่ใช้สาธิตวันนี้

### ใช้ได้

- `Synthetic model probability`
- ระดับสาธิต ต่ำ/ปานกลาง/สูง
- SHAP factor contribution ใน Raw-margin space
- Weather/Ocean/Sentinel-2 ในฐานะ Live data context
- ข้อความ `ข้อมูลไม่เพียงพอสำหรับ Operational prediction`

### ยังใช้ไม่ได้

- `ความเสี่ยงจริงของบางแสน = X%`
- `แม่นยำ X% ในพื้นที่จริง`
- `คาดว่าจะเกิด Bloom จริงภายใน 3–5 วัน`
- `ภาพดาวเทียมยืนยันชนิดแพลงก์ตอนแล้ว`

## 5. สิ่งที่ต้องทำหลังส่ง

1. กำหนดนิยาม Bloom label กับผู้เชี่ยวชาญ
2. เก็บ Verified ground truth ทั้ง Positive และ Negative events
3. จับคู่ข้อมูลที่รู้ ณ เวลา `t` กับ Outcome ณ `t+3..5`
4. ฝึก No-image model และ Full model แยกกัน
5. ทดสอบ Temporal/Spatial holdout และ Calibration
6. เปิด Operational probability เฉพาะเมื่อผ่าน Approval gate
